# Task Scoping

## Objective

Turn open-ended requests into an actionable implementation target.

## Workflow

1. Identify requested outcome, constraints, and non-goals.
2. Define a minimal first version that can be delivered safely.
3. Capture assumptions and mark project-specific unknowns as placeholders.
4. Confirm validation expectations using `LINT_COMMAND`, `TEST_COMMAND`, and `BUILD_COMMAND`.
5. Provide a concise execution outline.

## Placeholder Pattern

Use placeholder names consistently:

- `LINT_COMMAND`
- `TEST_COMMAND`
- `BUILD_COMMAND`
- `FORMAT_COMMAND`
